Description

What I Made:

Aiming to explore the process of network bending, as seen in Daniel Manz project ‘Brave’, I created a generative music project using Markov chains. While the program generates sound based on the ‘training data’ of a song, the project can also be considered as critical software as I’m exploring how small language models can be used as an alternative to AI.

How I Made It:

Markov Chains: Following a Coding Train Markov Chains tutorial, I used the code as my main technique for generating the music notes. As the tutorial used sentences as training data, I modified the code to use music notes from a midi json file. Overall the technique is the same as in Shiffman’s tutorial, where the training data is converted into ngrams, and then used to predict the next likely notes. Where the higher ngram number results in music notes more closely resembling the original song.

Song input & Midi: Claire de lune, by Claude Debussy, 1905. I downloaded the Midi file from Bitmidi and converted it into the correct json format on the Tone.js Converter. I used the Tone.js midi repository to understand how to work with the json data.

Tone.js: I incorporated the Tone.js library as a method to work with synths. I looked at various resources to understand how to work with the library, including the Getting Started page, API and Examples. All synths are directly from Tone.js Examples while the Piano Synth is a sampler found on Tone.js GitHub.

FFT Visualization (Animated Circles): Using a combination of predominantly Bardin’s Audio Spectrum Meter and tomethecollins code , I used FFT levels to animate my circles.

‘Network Bending’: Based on Daniel Manz’s project ‘Brave’, I created a simpler version with Markov Chains by including functionality to change the ngram value, note duration and delay.

Why I Made It :

Beginning to work on this project I looked into Sarte’s writing on the social theory of Serialty, and related concepts such as ‘bad faith’ and the Practico Inert, comparing how these concepts can be related to current AI technology. Specifically how the Practico Inert explains how the material we produce can result in rigid systems which eventually disadvantage us. Such as how the system of industrialisation has had many positive benefits, while at the same time resulting as a major contribution to climate change (Oxford Dictionary). The use of AI can also be related to Sarte’s opinion that these systems also shape us, where because of their size, it can be impossible to opt out of. This can be seen in the pressure to use AI in order to keep up with the labour market (Fleming, 2022), while at the same time having a negative effect on our cognitive and creative capacities.

While these concepts were difficult to visually communicate, I looked into the use of small language models and network bending as processes opposed to the rigid systems of dominant AI technology. Using Mark Sample’s essay on SLM’s and Daniel Manz use of Network Bending as an "active divergence"(Manz) from AI.

As someone with very little experience working with music, the project was interesting to explore how I could use smaller generative methods to create music. While having full control over the inputs and methods, compared to using AI software that relies on large, and mostly undocumented, datasets.
